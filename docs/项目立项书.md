## 1. 项目概述

### 1.1 项目背景

FamilyWall 是一个家庭/家族回忆记录与管理平台，灵感来源于实体照片墙。用户可以通过地图标记家庭聚餐位置，以照片墙和时光轴两种形式展示家庭回忆，还可以创建类似小红书的图文笔记。

### 1.2 项目目标

打造一个连接家庭成员、记录家族聚餐与回忆的私密社交平台。

### 1.3 核心价值

- **回忆可视化**: 地图 + 照片墙 + 时光轴多维度展示
- **家族凝聚**: 家庭/家族双层管理结构，强化家族认同感
- **隐私安全**: 家庭内部分享，家族选择性开放

## 2. 功能范围

### 2.1 MVP 功能 (第一版)

| 模块 | 功能 |
|------|------|
| 用户系统 | 注册、登录、登出、Token认证 |
| 家庭管理 | 创建家庭、邀请码生成/加入、成员管理 |
| 家族管理 | 创建家族、添加/移除家庭、家族管理员 |
| 地图标记 | 在地图上标记聚餐位置、经纬度存储 |
| 照片墙 | 瀑布流展示、点击放大 |
| 时光轴 | 按时间倒序展示、日期分组 |
| 笔记 | 图文笔记创建、编辑、删除、可见性设置 |
| 权限管理 | 家庭管理员、家族管理员、成员权限 |

## 3. 技术方案

### 3.1 技术选型

| 端 | 技术栈 | 版本 |
|----|--------|------|
| 前端 | Vue 3 + TypeScript + Vite | Latest |
| 状态管理 | Pinia | ^3.0 |
| UI框架 | TailwindCSS | ^4.0 |
| 后端 | PHP 8.2 + Laravel | ^11.0 |
| 数据库 | MySQL | 8.0 |
| 地图 | 高德地图 JS API | v2.0 |
| 图片存储 | 阿里云 OSS / 本地存储 | - |

### 3.2 项目结构

```
FamilyWall/                    # 前端项目
├── src/
│   ├── views/                 # 页面视图
│   ├── components/            # 组件
│   ├── stores/                # Pinia状态
│   ├── api/                   # API调用
│   ├── types/                 # TypeScript类型
│   └── assets/                # 静态资源

FamilyWall-backend/            # 后端项目
├── app/
│   ├── Http/                  # Controller/Middleware
│   ├── Models/                # 数据模型
│   └── Services/              # 业务逻辑
├── database/
│   └── migrations/            # 数据库迁移
└── public/                    # 入口文件
```
