## 1. 产品概述

### 1.1 产品背景

FamilyWall 是一个家庭/家族回忆记录与管理平台，用户可以通过地图标记家庭聚餐位置，以照片墙和时光轴两种形式展示家庭回忆，还可以创建类似小红书的图文笔记。

### 1.2 目标用户

- 20-50岁的家庭用户
- 注重家庭回忆的用户群体
- 有家族凝聚需求的用户

## 3. 详细功能需求

### 3.1 用户系统

#### 3.1.1 注册
```
流程:
1. 用户进入注册页
2. 输入邮箱 → 获取验证码
3. 输入验证码
4. 设置密码 (8-20位，包含大小写字母+数字) → 二次确认密码
5. 填写昵称 (必填, 2-20字符)
6. 选择头像 (可选)
7. 选择性别 (必填)
8. 选择生日 (必填)
9. 提交注册 → 跳转完善个人信息页

注册后引导:
- 完善个人资料 (可选)
- 推荐创建/加入家庭
- 引导流程可跳过

错误提示:
- 邮箱格式错误: 弹窗提示"请输入正确的邮箱地址"
- 验证码错误: 弹窗提示"验证码错误，请重新输入"
- 密码强度不足: 弹窗提示"密码需包含大小写字母和数字"
- 两次密码不一致: 弹窗提示"两次密码输入不一致"
- 邮箱已被注册: 弹窗提示"该邮箱已被注册"
```

#### 3.1.2 登录
```
登录方式:
1. 邮箱 + 密码登录
2. 验证码登录 (免密登录)
3. 微信一键登录 (第三方)

登录流程:
1. 进入登录页
2. 输入邮箱 + 密码
3. 提交登录
4. 验证成功 → 返回Token → 跳转首页
5. 验证失败 → 弹窗提示错误

错误提示:
- 邮箱未注册: 弹窗提示"该邮箱未注册，请先注册"
- 密码错误: 弹窗提示"密码错误，请重新输入"
- 账号被禁用: 弹窗提示"账号已被禁用，请联系客服"

第三方登录 (微信):
流程:
1. 点击微信登录
2. 唤起微信授权
3. 授权成功 → 绑定邮箱 (未绑定时)
4. 登录成功

Token机制:
- JWT Token
- 有效期: 7天
- 存储: localStorage
- 携带方式: Authorization: Bearer <token>
```

#### 3.1.3 登出
```
流程:
1. 点击"设置" → "退出登录"
2. 确认弹窗
3. 确认 → 清除Token → 跳转登录页
4. 保留本地缓存 (如浏览记录偏好)

确认弹窗内容:
┌─────────────────────────────────┐
│        确认退出                 │
│                                 │
│     确定要退出登录吗？          │
│                                 │
│   ┌─────────┐  ┌─────────┐    │
│   │  取消   │  │  确认   │    │
│   └─────────┘  └─────────┘    │
└─────────────────────────────────┘
```

#### 3.1.4 忘记密码
```
入口: 登录页 "忘记密码" 链接

流程:
1. 点击"忘记密码"
2. 输入注册邮箱
3. 获取验证码
4. 输入新密码 → 二次确认
5. 提交重置 → 跳转登录页

错误提示:
- 邮箱未注册: 弹窗提示"该邮箱未注册"
- 验证码错误: 弹窗提示"验证码错误"
- 验证码过期: 弹窗提示"验证码已过期，请重新获取"
```

#### 3.1.5 注销账号
```
入口: 设置 → 注销账号

注销条件:
- 账号无进行中的交易
- 无未完成的纠纷

注销流程:
1. 阅读注销须知弹窗
┌─────────────────────────────────────────┐
│          注销须知 (必读)                 │
├─────────────────────────────────────────┤
│ 1. 注销后所有数据将被清除               │
│ 2. 注销后无法恢复                       │
│ 3. 30天后才可重新注册                   │
│ 4. 需验证密码确认身份                   │
│                                         │
│  ☑ 我已阅读并同意注销                   │
│                                         │
│   ┌─────────────┐ ┌─────────────┐    │
│   │    取消     │ │   下一步    │    │
│   └─────────────┘ └─────────────┘    │
└─────────────────────────────────────────┘

2. 验证身份 (密码/邮箱验证码)
3. 确认注销弹窗
4. 确认 → 进入7天冷静期
5. 冷静期可撤回

错误提示:
- 验证失败: 弹窗提示"身份验证失败，请重试"
- 有进行中业务: 弹窗提示"请先完成进行中的业务"

注销后:
- 所有数据清除
- 邮箱释放 (30天后可重新注册)
- 无法恢复
```

### 3.2 家庭模块

#### 3.2.1 创建家庭
```
入口: 首页点击"创建家庭"按钮

流程:
1. 点击创建家庭
2. 填写家庭名称 (必填, 2-30字符)
3. 选择家庭头像 (可选)
4. 提交创建
5. 创建者自动成为家庭管理员
6. 跳转家庭详情页

规则:
- 每个用户最多创建 5 个家庭
- 家庭名称不能重复
```

#### 3.2.2 邀请码管理
```
生成邀请码:
1. 进入家庭详情页
2. 点击"邀请成员"
3. 系统生成6位邀请码
4. 可设置有效期 (必填，默认1天，最少1天)
5. 复制邀请码分享

邀请码规则:
- 格式: 6位字母数字组合 (如: ABC123)
- 有效期: 必填 1-30 天，默认1天
- 数量限制: 每天最多生成 30 个
- 状态: 未使用/已使用/已过期
- 支持批量生成

错误提示:
- 超过数量限制: 弹窗提示"今日生成数量已达上限(30个)，请明天再试"
```

#### 3.2.3 加入家庭
```
流程:
1. 点击"加入家庭"
2. 输入邀请码
3. 验证邀请码有效性
4. 成功 → 加入家庭 → 跳转家庭详情页
5. 失败 → 弹窗提示错误

验证:
- 邀请码是否存在: 弹窗提示"邀请码不存在"
- 邀请码是否过期: 弹窗提示"邀请码已过期，请联系管理员获取新邀请码"
- 是否已加入该家庭: 弹窗提示"您已是该家庭成员"
- 家庭成员是否已满: 弹窗提示"该家庭成员已满"
```

#### 3.2.4 家庭成员管理
```
成员角色:
- 管理员 (admin): 管理家庭、邀请成员、移除成员、编辑家庭信息
- 成员 (member): 查看、创建笔记、查看照片墙

查看成员列表:
- 成员昵称
- 成员头像
- 加入时间
- 角色标识

移除成员:
- 仅管理员可操作
- 不可移除自己(创建者)
- 移除后成员收到通知

转让管理员:
- 仅管理员可操作
- 选择新管理员
- 转让后原管理员变为成员
```

#### 3.2.5 家庭信息编辑
```
可编辑信息:
- 家庭名称 (仅管理员)
- 家庭头像 (仅管理员)

操作:
- 保存修改 → 立即生效
- 记录修改历史 (可选)
```

### 3.3 家族模块

#### 3.3.1 创建家族
```
入口: 家族页点击"创建家族"

流程:
1. 点击创建家族
2. 填写家族名称 (必填, 2-30字符)
3. 选择家族头像 (可选)
4. 提交创建
5. 创建者自动成为家族管理员
6. 跳转家族详情页

规则:
- 每个用户最多创建 1 个家族
- 家族名称全局唯一
```

#### 3.3.2 添加家庭到家族
```
两种方式:

方式一: 家庭主动申请
1. 家庭管理员进入家族页
2. 浏览可加入的家族
3. 点击申请加入
4. 等待家族管理员审核
5. 审核通过 → 加入成功

方式二: 家族管理员邀请
1. 家族管理员进入家族管理页
2. 点击"邀请家庭"
3. 输入家庭名称搜索
4. 发送邀请
5. 家庭管理员同意 → 加入成功

状态流转:
pending → approved / rejected
```

#### 3.3.3 家族管理
```
管理员功能:
1. 查看所有加入的家庭列表
2. 审核家庭的加入申请
3. 移除家庭 (需确认)
4. 查看家族笔记
5. 隐藏/显示家族笔记
6. 转让管理员

成员功能:
1. 查看家族信息
2. 查看家族照片墙
3. 查看家族笔记
```

#### 3.3.4 家族管理员
```
添加管理员:
- 现有管理员可添加新管理员
- 需选择用户(家庭管理员)

管理员权限:
- 所有成员权限
- 审核家庭加入
- 移除家庭
- 管理笔记 (隐藏/显示)
- 转让管理员

注意:
- 至少保留1个管理员
- 转让后原管理员变为普通成员
```

### 3.7 笔记模块

#### 3.7.1 创建笔记
```
入口: 首页点击"+"按钮 → 选择"笔记"

流程:
1. 点击创建笔记
2. 选择可见范围 (家庭/家族)
3. 填写标题 (必填, 1-100字符)
4. 填写正文 (必填, 支持富文本)
5. 上传图片 (可选, 最多50张, 每张最大10MB)
6. 上传视频 (可选, 最多5个, 每个最大100MB)
7. 关联位置 (可选, 从已有位置选择或新建)
8. 预览
9. 发布

富文本支持:
- 文字加粗/斜体/下划线
- 标题 (H1-H3)
- 列表 (有序/无序)
- 引用
- 链接

媒体处理:
- 图片: 上传前压缩，支持裁剪，添加水印(可选)
- 视频: 支持MP4/AVI格式，上传进度条，转码处理

错误提示:
- 标题为空: 弹窗提示"请输入标题"
- 正文为空: 弹窗提示"请输入正文"
- 图片数量超限: 弹窗提示"最多上传50张图片"
- 视频数量超限: 弹窗提示"最多上传5个视频"
- 文件过大: 弹窗提示"文件过大，请压缩后上传"
- 上传失败: 弹窗提示"上传失败，请重试"

  │         └──→ 注册 ──→ 首页
  │
  └──→ 首页 (部分功能受限)
```

### 5.2 家庭创建流

```
首页 ──→ 点击创建家庭 ──→ 填写信息 ──→ 创建成功 ──→ 家庭详情页
                                            │
                                            └──→ 自动成为管理员
```

### 5.3 加入家庭流

```
首页 ──→ 点击加入家庭 ──→ 输入邀请码 ──→ 验证成功 ──→ 加入成功 ──→ 家庭详情页
                                                                 │
验证失败 ←────────────────────────────────────────────────────┘
```

### 5.4 创建笔记流

```
首页 ──→ 点击+ ──→ 选择笔记 ──→ 选择可见范围 ──→ 填写内容 ──→ 上传图片 ──→ 预览 ──→ 发布
                    │                           │
                    └──→ 关联位置 (可选) ───────┘
```

### 5.5 家族申请流

```
家族页 ──→ 浏览家族 ──→ 申请加入 ──→ 等待审核 ──→ 审核通过 ──→ 加入成功
                                                        │
                                              审核拒绝 ←─┘
```

## 6. 数据模型

### 6.1 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    nickname VARCHAR(50) NOT NULL,
    avatar VARCHAR(500),
    gender ENUM('unknown', 'male', 'female') NOT NULL,
    birthday DATE NOT NULL,
    bio VARCHAR(200),
    status ENUM('active', 'banned', 'deleted') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 第三方登录表
CREATE TABLE social_accounts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    provider ENUM('wechat') NOT NULL,
    provider_user_id VARCHAR(100) NOT NULL,
    access_token TEXT,
    refresh_token TEXT,
    expires_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE KEY unique_provider_user (provider, provider_user_id)
);

-- 家庭表
CREATE TABLE families (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    avatar VARCHAR(500),
    invite_code VARCHAR(10) UNIQUE,
    invite_expire_at TIMESTAMP,
    creator_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (creator_id) REFERENCES users(id)
);

-- 家庭成员表
CREATE TABLE family_members (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    family_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    role ENUM('admin', 'member') DEFAULT 'member',
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE KEY unique_family_user (family_id, user_id)
);

-- 家族表
CREATE TABLE clans (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) UNIQUE NOT NULL,
    avatar VARCHAR(500),
    creator_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (creator_id) REFERENCES users(id)
);

-- 家族-家庭关联表
CREATE TABLE clan_families (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    clan_id BIGINT NOT NULL,
    family_id BIGINT NOT NULL,
    status ENUM('pending', 'approved', 'rejected') DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (clan_id) REFERENCES clans(id),
    FOREIGN KEY (family_id) REFERENCES families(id),
    UNIQUE KEY unique_clan_family (clan_id, family_id)
);

-- 家族管理员表
CREATE TABLE clan_admins (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    clan_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (clan_id) REFERENCES clans(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE KEY unique_clan_admin (clan_id, user_id)
);

-- 位置标记表
CREATE TABLE locations (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    family_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    name VARCHAR(200) NOT NULL,
    address VARCHAR(500),
    latitude DECIMAL(10, 8) NOT NULL,
    longitude DECIMAL(11, 8) NOT NULL,
    event_date DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 笔记表
CREATE TABLE notes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    family_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    location_id BIGINT,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    visibility ENUM('family', 'clan') DEFAULT 'family',
    is_hidden TINYINT(1) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (location_id) REFERENCES locations(id)
);

-- 笔记图片表
CREATE TABLE note_images (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    note_id BIGINT NOT NULL,
    url VARCHAR(500) NOT NULL,
    sort_order INT DEFAULT 0,
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE
);

-- 通知表
CREATE TABLE notifications (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    type VARCHAR(50) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    data JSON,
    is_read TINYINT(1) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 笔记评论表
CREATE TABLE note_comments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    note_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    parent_id BIGINT DEFAULT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP DEFAULT NULL,
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (parent_id) REFERENCES note_comments(id)
);

-- 笔记点赞表
CREATE TABLE note_likes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    note_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE KEY unique_note_user_like (note_id, user_id)
);

-- 用户关注表 (用于@功能)
CREATE TABLE user_follows (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    follower_id BIGINT NOT NULL,
    following_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (follower_id) REFERENCES users(id),
    FOREIGN KEY (following_id) REFERENCES users(id),
    UNIQUE KEY unique_follow (follower_id, following_id)
);
```
